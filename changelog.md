# Changelog

## VOID 1.4 改进说明

相较于 1.2 版本，当前的 `VOID` 插件已完成以下改进。

### 一、Typecho 兼容性与基础稳定性提升

- 从源码实现看，当前版本仍保留了对 Typecho 1.2 的兼容调用方式，同时补全了对 Typecho 1.3 接口与命名差异的适配。新增的接口别名与类名回退主要用于兼容 1.3，而旧式 `Typecho_*` / `Widget_*` 调用链并未被移除，因此现阶段可视为兼容 Typecho 1.2 与 1.3。
- 插件激活逻辑已增强为具备环境前置校验能力，不再仅依赖运行时暴露错误，而是主动检查 `OpenSSL`、`DOM` 扩展以及 IP 库初始化条件，并给出更明确的失败提示。
- 数据表、字段与索引的检测方式已从早期依赖 SQL 模糊匹配的实现，升级为更准确的结构判断，避免误判或升级逻辑不稳定的问题。
- 投票相关数据表的升级逻辑已有补强，除了兼容旧表结构，还新增了对 `created` 字段及相关索引的迁移处理，使历史安装升级后的表现更加稳定。

### 二、IP 定位能力全面升级

- 基于旧 `17monipdb.dat` 的归属地解析方案已退出主运行链路，当前版本统一切换至 `ip2region xdb` 方案。
- 新版本已实现 IPv4 与 IPv6 的统一支持，解决了旧实现面对现代网络环境时的明显短板。
- 针对 `127.0.0.1`、`::1`、私网地址、保留地址等特殊 IP，新版本提供了更稳定且更友好的返回结果，不再大量暴露含糊的 `unknown`、`N/A` 等历史文案。
- 评论管理与互动页面中涉及 IP 归属地展示的调用链已统一到新的定位服务，格式化逻辑也不再分散在页面层做额外替换，整体一致性更好。
- 相较于早期版本，这一部分不仅是更换数据库文件，而是完成了从历史 IPv4 本地库方案到现代可维护定位服务的整体替换。

### 三、图片尺寸解析链路优化

- 早期版本的图片尺寸解析依赖 `simple_html_dom`，并通过较旧的 HTML 解析方式提取文章图片信息；当前版本已移除这一依赖，改为使用原生 `DOMDocument` 与 `DOMXPath`。
- 新版本保留了原有的 `#vwid` / `#vhei` 尺寸标记机制，以确保早期文章内容中的历史数据仍可继续生效，不破坏既有内容兼容性。
- 图片解析逻辑已被拆分为更清晰的处理流程，包括内容转 HTML、图片源提取、尺寸回写、内容统一更新等步骤，代码结构比早期实现更清楚。
- 早期版本在解析过程中可能按图片逐次写回数据库；当前版本改为一次性处理后统一更新文章内容，减少了不必要的数据库写入。
- 新版本对空内容、无图内容、DOM 不可用等边界情况增加了防御性处理，运行稳定性优于早期版本。
- 结合主题前端后续配套调整，当前链路对普通图片、懒加载图片与图集场景的兼容性也更完整。

### 四、UA 解析能力明显增强

- 早期版本的 UA 解析规则较为简单，浏览器与系统识别范围有限，部分命名、拼写和展示结果也不够规范。
- 当前版本重构了 UA 解析逻辑，对浏览器识别规则进行了明显扩展，已覆盖 Edge、Opera、Vivaldi、Yandex、Samsung Internet、QQ 浏览器、UC、微信、Chrome、Safari、Firefox、IE 等常见客户端。
- 系统识别能力也得到补强，已可更好识别 Windows、Android、iOS / iPadOS、macOS、HarmonyOS、ChromeOS 以及常见 Linux 发行版。
- 除传统浏览器外，新版本也能够更准确地识别 `curl`、`Wget`、`python-requests`、`Postman`、`OkHttp` 等非浏览器客户端，有助于后台日志和互动信息展示更准确。
- UA 输出结构相较早期版本更统一，浏览器名称、版本号、系统信息与图标映射的组织方式更加规范，展示质量整体提升。

### 五、字数统计逻辑升级

- 早期版本的字数统计实现相对基础，主要聚焦于中文字符与英文词数量的简单计算。
- 当前版本已将字数统计扩展为更完整的内容分析能力，在统计前会主动清理 `#vwid` / `#vhei` 标记、`<!--more-->`、Markdown 图片语法、短代码、表情标记、围栏代码块、行内代码等容易干扰统计结果的内容。
- 新版本将统计结果拆分为中文字符数、英文单词数、数字数量等多个维度，统计结果比早期版本更细，也更适合后台展示。
- 统计逻辑已支持文章与页面场景，并增加了对空值、异常内容、复杂 HTML / Markdown 混排场景的处理。
- 这一改动使插件内部的统计能力不再只是附带功能，而是已经具备替代部分独立字数统计插件的实际价值。

### 六、互动接口与后台实现更稳健

- 早期版本的互动接口虽然可用，但参数校验、错误返回、重复操作判定等方面偏历史实现风格；当前版本已对这一部分做了系统性整理。
- 新版本为点赞、赞踩等交互增加了更明确的参数校验、目标存在性检查与标准化 JSON 错误返回，接口行为更加可预期。
- 请求校验逻辑已有增强，补充了 POST 校验、Typecho token 校验以及同源校验等保护措施，提升后台及接口安全性。
- 重复投票判定与异常分支处理已比早期版本更清晰，减少了旧版请求命中但状态不明确的情况。
- 后台互动列表的展示逻辑已配合新的 IP 与 UA 服务做统一接入，页面信息质量和稳定性较早期版本更好；旧的 `showActivity.php` 及其配套历史资源也已退出主路径。
- 互动相关数据库结构在升级过程中也同步做了补强，包括时间字段与索引优化，为后续数据量增长提供了更好的基础。

### 七、后台编辑器能力增强

- 当前版本新增了后台写作时的实时统计面板，这是早期插件中不存在的能力。
- 该面板可在文章或页面编辑过程中实时显示字数、图片数、标签数等信息，显著提升写作和校对体验。
- 这部分功能使插件从偏前台增强、偏后台互动工具，进一步扩展为对内容生产流程也有帮助的综合型插件。
- 新版本还对移动端后台展示进行了适配调整，保证该面板在较窄屏幕下仍具备可读性和可操作性。
